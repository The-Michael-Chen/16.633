Introduction to Image Computations: Object Tracking Part 3

Author

Dominic Maggio

1 Today

As promised, this week’s content will be on the shorter side to give everyone time to catch-up. Next week we will begin discussing how to design a controller for our drones. For example, simple tasks for our controller might be to position the drone such that the tag is in the center of the image, or move our drone such that it is a desired distance from the tag. As this will require your tag tracker, it is important that you have your tracker working in time for next week’s class.

Today, we will make our tag tracker code from last week more modular and create a wrapper function for it so that it will be easier for us to integrate into our overall autonomy pipeline later.

Today, create a function with the following api

detect_tags(img, target_point_dist = None, visualize = False)
INPUTS:
img: input image that may contain any number of tags (including no tags)

target_point_dist: distance in meters below the
y direction of the tag to compute a point. Do not compute if None.

visualize: used for debugging. if True, visualize the tag
detection for all tags (ie show an img with a bounding box
for all detected tags in the image)

RETURNS:
a dictionary of the following form:
{tag_number:{{tag_id:tag, 'target':
{'position':p_camera_tag, 'pixel_coords':(px,py)}}}

where tag is an object returned by the april tags library described here https://pupil-apriltags.readthedocs.io/en/stable/api.html.

target is None if target_point_dist is None, otherwise, p_camera_tag is the 3D position of the target point in the camera’s frame and (px, p_y) gives the 2D pixel coordinates of the target point.
